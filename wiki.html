Overview

What NightFlix Does

NightFlix is a SwiftUI media browser that uses TMDB for metadata and an optional user-provided playback provider for embedded viewing.

Browse metadata

The home, search, category, movie detail, and series detail screens are powered by TMDB endpoints for titles, posters, backdrops, genres, cast, trailers, and recommendations.

Keep local lists

My List and Continue Watching are saved locally on the device. Clearing setup details removes the TMDB token and provider URL from local app storage.

Use your own provider

NightFlix does not ship a movie catalog or host streams. If playback is enabled, the provider URL must point to a lawful source you control or are allowed to use.

Setup

First-Time Setup

The app asks for a TMDB credential first. The playback provider can be configured during setup or later from Settings.

Create or open a TMDB account

Use the TMDB sign-up page if you do not already have an account. TMDB recommends completing API registration from a desktop browser.

Copy the API Read Access Token

NightFlix expects the long TMDB API Read Access Token and sends it in the Authorization: Bearer header. The app accepts the token with or without the leading word Bearer.

Add a legal provider URL

Enter only a provider that you are allowed to embed. You can skip this step and add it later in Settings, but playback requires a configured provider.

Open NightFlix

After setup, the app loads TMDB-powered rows such as popular movies, popular TV series, top-rated titles, categories, details, recommendations, and search results.

TMDB Key

How to Create a TMDB Key

TMDB shows both a v3 API key and a newer API Read Access Token. NightFlix is built for the API Read Access Token.

Sign in to TMDB

Go to the TMDB sign-up page, create an account, verify your email if asked, and sign in.

Open API settings

From your profile menu, open account settings and choose API. You can also try the API settings page after signing in.

Request an API key

Accept TMDB's API terms, then request a key. If TMDB asks for the type, choose the developer option for a personal or test app and fill in the required app details honestly.

Copy the API Read Access Token

In the API settings area, copy the value labeled API Read Access Token. Paste that into NightFlix where the setup screen asks for TMDB Read Access Token.

Keep it private

Treat the token like a password. Do not commit it to source control, paste it into issue trackers, or ship a public build with your personal token baked into the app.

Use the token, not the v3 query key

TMDB's docs describe the API Read Access Token as a Bearer token that can be sent in an Authorization header. NightFlix does exactly that for calls to api.themoviedb.org.

A valid token usually looks like three long text segments separated by dots. That is why the app validates the credential by checking for three dot-separated parts.

Source Policy

Movie Provider Safety Rules

TMDB provides metadata. It does not provide playable movie streams for this app. Provider URLs need a separate legal and security review.

Do not use vidking.net

vidking.net as a movie source is maliciously known for piracy. Please do not use it. Do not document it as a recommended provider, do not hard-code it, and do not ask users to paste it into setup.

Playback providers are your responsibility

Use only providers where you have permission to access, embed, and distribute the content. Avoid domains associated with piracy, malware, invasive ads, credential theft, or unclear rights.

Approved source checklist

  • The provider is owned by you, your organization, or a licensed vendor.
  • The content rights are documented for the users and regions you support.
  • The provider supports HTTPS and does not redirect through unknown ad networks.
  • The provider can be removed quickly if it becomes unsafe or non-compliant.

Avoid these sources

  • Sites marketed around free access to paid movies or shows without authorization.
  • Domains reported for piracy, malware, forced redirects, deceptive ads, or popups.
  • Sources that obscure ownership, hosting location, or content licensing terms.
  • Any provider that asks users for unrelated credentials, extensions, or downloads.
Usage

Using the App

The main user flow is intentionally simple: discover titles, inspect details, save favorites, and open the configured provider when playback is available.

Area What it does Notes
Home Shows hero content and rows for popular and top-rated titles. Requires a valid TMDB Read Access Token.
Search Searches movies and TV series through TMDB's multi-search flow. Results are filtered to movie and TV media types.
Categories Loads movie and TV genres, then discovers titles by genre. Genre lists are cached longer because they rarely change.
Details Shows overview, cast, trailer links, seasons, episodes, similar titles, and recommendations. TV playback needs season and episode numbers.
Settings Lets you update the TMDB token, provider URL, appearance, animation, cache, and setup state. Use this if setup details change later.
Troubleshooting

Common Fixes

Most issues are either a missing TMDB token, an invalid provider URL, or a provider page that fails to load inside the embedded web view.

TMDB rows do not load

Reopen Settings, confirm the token is the API Read Access Token, and paste it without extra whitespace. If the app says authentication failed, generate or copy the token again from TMDB.

Play button cannot open a title

Add a provider URL in Settings. NightFlix builds playback URLs from that base URL and the selected TMDB title, season, and episode information.

Provider page fails to load

Confirm the provider is online, supports HTTPS, and allows embedded loading. If it redirects to suspicious pages, remove it and choose a safer provider.

Old metadata appears

Use the cache controls in Settings to clear cached TMDB data. NightFlix will download fresh metadata the next time it needs that endpoint.

Developer Notes

How the Pieces Fit

These notes are for anyone extending the app or reviewing the provider and credential flow.

1. Setup User enters token

The token and provider URL are normalized before being saved to app settings.

2. Metadata TMDBService

Requests call api.themoviedb.org with Authorization: Bearer.

3. UI SwiftUI screens

Home, search, category, detail, list, and settings views consume TMDB models.

4. Playback URL builder

The provider base URL is combined with an embed route for the chosen title.

5. Web view PlayerView

The generated URL opens in the embedded player with load and error states.

Generated provider routes

Movie route:

/embed/movie/{tmdbId}?color=e50914&autoPlay=true&nextEpisode=true&episodeSelector=true

TV route:

/embed/tv/{tmdbId}/{season}/{episode}?color=e50914&autoPlay=true&nextEpisode=true&episodeSelector=true

Files worth knowing

TMDBConfig.swift stores TMDB image bases and credential checks. TMDBService.swift performs metadata requests and caching. NightFlixStyle.swift includes settings normalization and the provider URL builder. PlayerView.swift loads the final embedded URL.
